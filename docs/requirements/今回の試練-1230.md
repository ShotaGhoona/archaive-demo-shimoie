# 今回の試練 - 図面一覧ページのデザイン実装

## 概要

このタスクでは、Claude Code のカスタムコマンド `/init-design-v1` を使いながら、図面一覧ページのデモUIを作成してもらいます。また、テーブル設定機能の最適なUIUXを考えて提案してください。

---

## タスク0: カスタムコマンドのセットアップ

このリポジトリはClaude Code用に作られていますが、Codex CLIを使う場合はファイルの配置場所が異なります。

### Claude Code と Codex CLI の違い

| 項目 | Claude Code | Codex CLI |
|------|-------------|-----------|
| プロジェクト用 | `.claude/commands/` | なし |
| 個人用 | `~/.claude/commands/` | `~/.codex/prompts/` |
| サブディレクトリ | 対応 | 非対応 |
| チーム共有（Git管理） | 可能 | 不可 |

### やること

`.claude/commands/init-design-v1.md` を `~/.codex/prompts/` にコピーしてください。

```bash
cp .claude/commands/init-design-v1.md ~/.codex/prompts/
```

これで Codex CLI から `/init-design-v1` が使えるようになります。

---

## タスク1: 図面一覧ページのデモUI作成

`/init-design-v1` コマンドを使って、FSD準拠の図面一覧ページを作成してください。

### 作成するもの

| 要素 | 説明 |
|------|------|
| アプリヘッダー | 図面管理ページ以外は作らなくてOK。ただし存在はさせる |
| ダミーデータ | 図面データのモックを作成 |
| テーブルビュー | 図面の一覧をテーブル形式で表示 |
| ギャラリービュー | 図面のサムネイルをカード形式で表示 |
| フィルターサイドバー | サイドバーでフィルター条件を選択 |
| 検索窓 | キーワード検索 |
| ビュースイッチ | テーブル/ギャラリーの切り替えボタン |

### 進め方

**Step 1:** Claude Code で `/init-design-v1` を実行

**Step 2:** 自由に作る。FSDのpage-components層とwidgets層の理解を深めながら進めてください！

**Step 3:** `/init-design-v1` について気になった点があればメモしておく
